Safety guide

Security & Privacy

Creating a fine-grained Personal Access Token

  1. Go to GitHub → Settings → Developer settings → Personal access tokens → Fine-grained tokens
  2. Click Generate new token
  3. Set an expiry — 7 or 30 days is recommended; never "no expiry"
  4. Under Repository access, select Only select repositories → choose your takhtescore repo only
  5. Under Permissions → Repository permissions, set Contents → Read and write. Grant nothing else.
  6. Click Generate token and copy it immediately

How this page keeps your token safe

  • sessionStorage only — the token lives in sessionStorage, which is wiped automatically when you close the tab or browser window. It never touches localStorage, cookies, or any server.
  • Never sent anywhere except api.github.com — the token appears only in the Authorization header of direct calls to the GitHub REST API. No analytics, no third-party scripts.
  • Fine-grained scope — the token you create above can only read/write the contents of one specific repository. Even if it were stolen, it cannot access any other repo, org, or account setting.
  • Short expiry — a 7-day token that leaks is far less damaging than a permanent one. Set a calendar reminder to rotate it.
  • noindex header — this page has <meta name="robots" content="noindex, nofollow"> so search engines won't index it or link to it.
  • No token in URLs — the token is never placed in query strings or hash fragments that could appear in server logs or browser history.
  • "Forget token" button — clears sessionStorage immediately without needing to close the tab.

Additional recommendations

  • Use admin.html locally — bookmark it from a local server (npx serve .). You don't need to visit it on the live GitHub Pages URL.
  • Optionally add admin.html to .gitignore — if you'd rather not publish this page publicly at all, add admin.html to .gitignore and only keep it on your local machine. The site works fine without it.
  • Never share a screenshot of the page while a token is entered — the password field masks it, but be careful.
  • Revoke immediately if you suspect exposure — GitHub → Settings → Developer settings → Fine-grained tokens → Revoke.
