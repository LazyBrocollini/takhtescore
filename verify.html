How Verification Works

        

          

            
1

            

              
Load the ledger

              
Fetch data/matches.json directly from GitHub Pages — no intermediary server involved in reads.

            

          

          

            
2

            

              
Find the record

              
Locate the record by Match ID. If missing, the record was never certified or has been removed.

            

          

          

            
3

            

              
Recompute match hash

              
Reconstruct the canonical JSON (deterministic key order) and recompute SHA-256 in your browser using the Web Crypto API.

            

          

          

            
4

            

              
Verify the chain

              
Find the previous match in the player pair's chain, verify its chain hash matches the stored previous_hash, then recompute the chain hash.

            

          

          

            
5

            

              
Report result

              
If all hashes match: certified. Any mismatch indicates the ledger or a prior record has been altered.
